## Parte I - Introdução ao terminal

### Se localizando no terminal

Ao entrar no terminal, você verá algo similar a isto:

```bash
usuario@DESKTOP-00RFJVC:~$
````

Em verde está indicado o nome do seu computador ou servidor. Em vermelho, o caminho (ou endereço) no qual você se encontra. O símbolo `~` representa o caminho para seu diretório de usuário (também reconhecido por `/home`). Para checar o diretório atual no qual você se encontra, use o comando `pwd`.

```bash
usuario@DESKTOP-00RFJVC:~$ pwd
/home/user
```

Para checar os diretórios e arquivos contidos no diretório em que você se encontra, use o comando `ls` (*list*).

```bash
usuario@DESKTOP-00RFJVC:~$ ls
```

Visualize informações adicionais utilizando os argumentos `-l` (retorna os resultados em lista), `-s` (tamanho dos arquivos), `-h` (tamanhos em formato mais compreensível) e `-a` (mostra arquivos ocultos, isto é, cujo nome começa com `.`).

Dentro da pasta dados, liste as pastas existentes usando:

```bash
usuario@DESKTOP-00RFJVC:~$ ls -l -h -a
```

Exemplo de saída:

### Criando diretórios

Para criar um novo diretório, temos o comando `mkdir` (*make directory*). Para usá-lo, basta adicionar o nome do diretório (não pode conter espaços!) depois do comando.

Crie um diretório com o nome `pratica1`:

```bash
usuario@DESKTOP-00RFJVC:~$ mkdir pratica1
```

No Linux, não se recomenda o uso de espaço, acentos e cedilha em títulos de arquivos ou diretórios. Caso realmente precise criar um diretório com espaço no título, use aspas (ex: `"nome do diretório"`). Ainda assim, dê preferência ao underline (`_`) ou outro caractere.

Para entrar nesse diretório novo, use o comando `cd` (*change directory*), apenas indicando-o depois do `cd`.

```bash
usuario@DESKTOP-00RFJVC:~$ cd pratica1/
```

Para ir ao diretório superior (isto é, o diretório no qual o diretório atual está contido), basta indicar `..`.

```bash
usuario@DESKTOP-00RFJVC:~/pratica1$ cd ..
```

Caso tenha alguma dúvida de como ou quais argumentos utilizar para determinado comando, você pode utilizar o argumento `--help`. Experimente fazer isso com os comandos indicados.

Agora, entre no seu diretório contido na pasta `pratica1` para as próximas etapas desta atividade.

### Criando e editando arquivos de texto

Para criação de um novo arquivo, podem ser utilizados os comandos `touch` ou `nano`. O comando `nano` abrirá o arquivo para edição, enquanto o comando `touch` somente criará o arquivo.

Execute:

```bash
usuario@DESKTOP-00RFJVC:~$ touch text.txt
```

Para editar o arquivo, execute o comando `nano` no terminal:

```bash
usuario@DESKTOP-00RFJVC:~$ nano text.txt
```

Será aberto um documento em branco. Digite o seguinte texto:

```txt
String1~
String2>
string3
string 4
S t r i n g 6
String7String8String9
String*10
```

Para sair do documento pressione `Ctrl + x` e, em seguida, a tecla `Y` para salvar a edição e `Enter` para sobrescrever o arquivo (caso deseje, no futuro, criar um novo arquivo a partir da edição feita, aqui o nome do arquivo deve ser modificado).

Para ver o arquivo de texto criado na tela do terminal, use o comando `cat`.

```bash
usuario@DESKTOP-00RFJVC:~$ cat text.txt
String1~
String2>
string3
string 4
S t r i n g 6
String7String8String9
String*10
```

Para encontrar determinada sequência de caracteres alvo (também chamadas de *strings*), deve-se utilizar o comando `grep`, o qual realizará um print das linhas que contenham os dados de busca.

Por exemplo, para encontrar apenas as linhas que contenham o texto `"String"` no arquivo criado anteriormente (`text.txt`), deve-se executar:

```bash
usuario@DESKTOP-00RFJVC:~$ grep "String" text.txt
String1~
String2>
String7String8String9
String*10
```

> **Atenção:** Observe que foram identificados somente os termos que iniciavam com letra maiúscula (`String`). Isto ocorre porque o sistema diferencia, em suas buscas e linhas de comando, caracteres maiúsculos e minúsculos.

---

### Substituindo strings com `sed`

Para substituir *strings* de um arquivo, utilizaremos o comando `sed`. O comando utiliza, respectivamente, a string que deseja-se procurar e aquela que irá substituir o termo.

Execute:

```bash
sed 's/ring/outra coisa/' text.txt
```

O resultado impresso na tela do terminal será:

```txt
Stoutra coisa1~
Stoutra coisa2>
stoutra coisa3
stoutra coisa 4
S t r i n g 6
Stoutra coisa7String8String9
Stoutra coisa*10
```

Para sobrescrever o arquivo com o resultado gerado, utilize o argumento `-i`:

```bash
sed -i 's/ring/outra coisa/' text.txt
```

Repare, contudo, que dois `ring` não foram substituídos. Isto ocorreu porque o comando `sed`, por padrão, substitui somente a primeira string de cada linha.

Para que todos os termos `ring` escritos sejam substituídos por `outra coisa`, basta inserir um `g` ao fim do argumento com as aspas.

```bash
sed -i 's/ring/outra coisa/g' text.txt
```

---

### Executando dois comandos simultaneamente

É possível aplicar dois comandos distintos em uma única execução. Por exemplo, para utilizar o comando `grep` a partir do resultado gerado pelo `sed`, basta adicionar `|` entre os comandos:

```bash
usuario@DESKTOP-00RFJVC:~$ sed 's/ring/outra coisa/' text.txt | grep 'S'
```

Saída:

```txt
Stoutra coisa1~
Stoutra coisa2>
S t r i n g 6
Stoutra coisa7String8String9
Stoutra coisa*10
```

Para salvar essa saída em um arquivo, use `>` (“direcionar”) seguido do nome do novo arquivo.

```bash
usuario@DESKTOP-00RFJVC:~$ sed 's/ring/outra coisa/' text.txt | grep 'S' > nomequeeuquero.txt
```

Olhe o conteúdo do arquivo:

```bash
usuario@DESKTOP-00RFJVC:~$ cat nomequeeuquero.txt
Stoutra coisa1~
Stoutra coisa2>
S t r i n g 6
Stoutra coisa7String8String9
Stoutra coisa*10
```

Para realizar uma nova busca e adicionar o resultado ao final do arquivo anterior, utilize `>>` ao invés de `>`:

```bash
usuario@DESKTOP-00RFJVC:~$ sed 's/ring/outra coisa/' text.txt | grep '~' >> nomequeeuquero.txt
```

```bash
usuario@DESKTOP-00RFJVC:~$ cat nomequeeuquero.txt
Stoutra coisa1~
Stoutra coisa2>
S t r i n g 6
Stoutra coisa7String8String9
Stoutra coisa*10
Stoutra coisa1~
```

### Copiando, movendo e renomeando arquivos e pastas

Para copiar um arquivo, use o comando `cp`, indicando o nome do arquivo a ser copiado e o nome que a nova cópia terá.

```bash
usuario@DESKTOP-00RFJVC:~$ cp nomequeeuquero.txt copia_nome.txt
```

Para copiar para outro diretório, adicione o caminho desse diretório antes do nome da cópia.

```bash
usuario@DESKTOP-00RFJVC:~$ mkdir new_dir
usuario@DESKTOP-00RFJVC:~$ cp nomequeeuquero.txt new_dir/copia2.txt
```

O comando `mv` pode também ser utilizado para renomear arquivos e pastas, movendo-os para o mesmo diretório com outro nome.

```bash
usuario@DESKTOP-00RFJVC:~$ mv copia_nome.txt new_dir/copia3
```

Para remover arquivos, use o comando `rm` seguido do nome do arquivo que deseja excluir.

```bash
usuario@DESKTOP-00RFJVC:~$ rm new_dir/copia3
```

> **Importante:** O sistema Linux não alerta ou pede confirmação de exclusão de seus arquivos. Portanto, arquivos excluídos não vão para lixeira ou pastas temporárias e, logo, são perdidos definitivamente, sem possibilidade de recuperação. Sempre tenha um backup dos seus arquivos!

### Organizando e filtrando arquivos

Nesta etapa, vamos utilizar uma tabela disponível online para explorar algumas formas de facilitar a visualização e edição de arquivos.

Uma tabela contendo o mapa de inserção de transposons em *Drosophila* (FlyBase) pode ser encontrada no seguinte link:

<a href="https://s3ftp.flybase.org/releases/FB2021_01/precomputed_files/insertions/insertion_mapping_fb_2021_01.tsv.gz"
   target="_blank" rel="noopener noreferrer">
  insertion_mapping_fb_2021_01.tsv.gz
</a>

Para download do arquivo direto do terminal, use o comando `wget`:

```bash
usuario@DESKTOP-00RFJVC:~$ wget http://ftp.flybase.org/releases/FB2021_01/precomputed_files/insertions/insertion_mapping_fb_2021_01.tsv.gz
```

O arquivo está em uma forma compactada (`.gz`). Dentre as diversas ferramentas para descompactar arquivos, usaremos `gunzip`:

```bash
usuario@DESKTOP-00RFJVC:~$ gunzip insertion_mapping_fb_2021_01.tsv.gz
```

O comando `cut` pode ser usado para extrair da tabela apenas as colunas desejadas:

```bash
cut -fX FILE
```

Onde `X` é o número da coluna desejada e `FILE` é o arquivo em questão.

Faça um teste extraindo a terceira coluna do arquivo e a salvando em um novo arquivo:

```bash
usuario@DESKTOP-00RFJVC:~$ cut -f3 insertion_mapping_fb_2021_01.tsv > col3
```

Para visualizar somente as primeiras 10 linhas do arquivo, execute o comando `head`:

```bash
usuario@DESKTOP-00RFJVC:~$ head col3
```

Exemplo de saída:

```txt
## FlyBase Insertion Mapping Table
## Generated: Tue Feb  2 21:30:51 2021
## Using datasource: dbi:Pg:dbname=fb_2021_01_reporting;

genomic_location
2L:19044446..19044446

2R:24123251..24123251
```

Remova as linhas em branco e, em seguida, o cabeçalho, com o comando `sed` (para melhor entendimento, leia o manual do comando):

```bash
usuario@DESKTOP-00RFJVC:~$ sed '/^$/d' col3 > col3a
usuario@DESKTOP-00RFJVC:~$ sed '/#/d' col3a > col3b
```

Para ordenar os resultados em ordem alfabética, você pode usar `sort`:

```bash
usuario@DESKTOP-00RFJVC:~$ sort col3b > sort_col3
```

> **Observação:** Caso tente ordenar o arquivo original com múltiplas colunas, não será possível usar o comando `sort` para ordenar a tabela inteira com base em uma única coluna que não seja a primeira.

Para remover linhas repetidas adjacentes, aplique o comando `uniq` (razão pela qual o comando `sort` é utilizado antes, garantindo que linhas idênticas estejam adjacentes na tabela):

```bash
usuario@DESKTOP-00RFJVC:~$ uniq sort_col3 > uniq_col3
```

Por fim, para comparar o número de linhas dos arquivos gerados, utilize o comando `wc` e o argumento `-l` (*lines*; ver `help`):

```bash
usuario@DESKTOP-00RFJVC:~$ wc -l sort_col3 uniq_col3
70770 sort_col3
63635 uniq_col3
134405 total
```

### FASTQ_SCREEN

Para instalar o FastQ Screen, abra o GitHub em:

* [https://github.com/StevenWingett/FastQ-Screen/releases/tag/v0.15.2](https://github.com/StevenWingett/FastQ-Screen/releases/tag/v0.15.2)

Copie o código do arquivo `.tar.gz`. Utilize o comando `wget` para baixar o arquivo e descompacte com o `gunzip`. Ao final, o executável `fastq_screen` deve estar dentro da pasta `FastQ-Screen-0.14.1`.

Para rodar o `fastq_screen`, o `bowtie2` terá que ser instalado também:

```bash
sudo apt-get install bowtie2
```

O `fastq_screen` é utilizado para mapear as reads contra vários genomas diferentes. Isso é feito para identificar possíveis contaminações e trocas de amostras.

Mapeando contra uma referência, podemos saber as possíveis origens das reads (isto é, se elas pertencem à amostra esperada ou se vêm de possíveis contaminantes) antes de prosseguir com as análises. Esse mapeamento é feito através de alinhamentos por similaridade usando softwares específicos independentes do `fastq_screen`, como o `bowtie2`.

## Parte III - Filtragem de datasets (VCF, Variant Call Format)

Nesta etapa, trabalharemos com um arquivo em formato VCF. Este formato é utilizado para armazenar informações detalhadas sobre as variantes identificadas.

Cada variante é também chamada de **SNP** (*Single Nucleotide Polymorphism*), e caracteriza uma base nucleotídica que varia entre os indivíduos analisados.

Veja o exemplo a seguir:

```txt
Indivíduo 1  ACGAACGTGTACAC
Indivíduo 2  ACGAAGGTGTACAC
Indivíduo 3  ACGAACGAGTACAT
Indivíduo 4  ACGAAGGTGTACAG
```

Observe que 4 indivíduos sequenciados para uma determinada região genômica apresentam 3 pares de bases que diferem entre si e, portanto, 3 SNPs podem ser contabilizados.

Agora veja:

```txt
Indivíduo 1  A-CGAAGGTACA
Indivíduo 2  AACGAAGGTACA
Indivíduo 3  AACGAAGGTACGA
```

Dos 4 indivíduos sequenciados, foram identificadas:

Diferentes dos SNPs, tais mutações são conhecidas como **INDELs** (*insertions/deletions*). As INDELs podem ser de uma ou mais bases.

### Filtrando o dataset

Nesta etapa, faremos a filtragem do dataset em `pulmao_Hs.vcf` com o programa `vcftools`.

Na genômica, as filtragens realizadas e os respectivos valores de corte dependerão da pergunta e do objetivo a ser atingido. As abordagens mais comuns incluem remoção de sequências com:

* baixa qualidade de leitura
* baixa cobertura
* alta taxa de *missing data* (dados faltantes/ausentes)

Faça o download e instalação do `vcftools` através das instruções apresentadas no link:

* [https://github.com/vcftools/vcftools](https://github.com/vcftools/vcftools)

Utilize o comando `git clone` para download e siga as instruções do site para instalação (tópico **“Build from GitHub”**).

Caso esteja utilizando o espaço do servidor fornecido para a disciplina, pule esta etapa de instalação.

Veja o manual do programa com a opção `man` (`man vcftools`) ou acesse:

* [http://vcftools.sourceforge.net/man_latest.html](http://vcftools.sourceforge.net/man_latest.html)

Para instalar, execute os comandos abaixo:

```bash
git clone https://github.com/vcftools/vcftools.git
cd vcftools
./autogen.sh
./configure
make
make install
```
